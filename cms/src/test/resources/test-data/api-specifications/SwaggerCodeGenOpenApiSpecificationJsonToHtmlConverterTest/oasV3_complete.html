Overview

Use this API to access the Personal Demographics Service (PDS)

Legal use

This API can only be used where there is a legal basis.

Search for patient

get
/Patient
Try this API

Endpoint

Use this endpoint to search for a patient in PDS.

Request

Query parameters
Name Description
_exact-match
Boolean
Default value: false
The search only returns results where the score field is 1.0.
Example: true
_history
Boolean
Default value: false
The search looks for matches in historic information such as previous names and addresses.
Example: true
Headers
Name Description
NHSD-Session-URID
String
A User Role ID (URID), also known as a User Role Profile ID (URPID), specifies the role the user is acting in.
Pattern: /^[0-9]+$/
Example: 555021935107
Authorization
String (^Bearer\ [[:ascii:]]+$)
An OAuthV2 access token presented in Bearer format.

Note: This parameter is required unless interacting with the Sandbox.

Example: Bearer g1112R_ccQ1Ebbb4gtHBP1aaaNM

Response

HTTP status: 200
A completed search, containing zero, one, or many matching patients.
Body
Content type: application/fhir+json
Example
{
  "resourceType" : "Bundle",
  "type" : "searchset",
  "timestamp" : "2019-12-25T12:00:00+00:00",

    "fullUrl" : "https://beta.api.digital.nhs.uk/personal-demographics/Patient/9000000009",
    "search" : {
      "score" : 1
    }
  } ]
}

Update a patient's resource

patch
/Patient/{id}
Try this API

Overview

Use this endpoint to update patient details in PDS.

Request

Path parameters
Name Description
id
String
The patient's NHS Number. The primary identifier of a patient, unique within NHS England and Wales.
Example: 9000000009
Required
Headers
Name Description
NHSD-Session-URID
String
A User Role ID (URID), also known as a User Role Profile ID (URPID).
Pattern: /^[0-9]+$/
Example: 555021935107
If-Match
String
Latest known version identifier enclosed in quotes preceded by W/.

Send the value of this resource's ETag response header (or meta.versionId property) as received.

Pattern: /^W\/"[0-9]+"$/
Example: W/"2"
Required
Body
Required
Content type: application/json
Examples
Add a new single item (deceasedDateTime) to the patient
{
  "patches" : [ {
    "op" : "add",
    "path" : "/deceasedDateTime",
    "value" : "2010-10-22T00:00:00+00:00"
  } ]
}
Update the simple item (gender)
{
  "patches" : [ {
    "op" : "replace",
    "path" : "/gender",
    "value" : "male"
  } ]
}

Endpoints: Polling

Retrieve the outcome of the accepted update.

get
/Polling/{message_id}
Try this API

Overview

Use this endpoint to poll for the outcome of an updated patient record.

Request

Path parameters
Name Description
message_id
String
The message ID of the accepted update that needs to be submitted to confirm the resource a successfully updated.
Example: 20200522091633363041000001
Required
Headers
Name Description
NHSD-Session-URID
String
A User Role ID (URID), also known as a User Role Profile ID (URPID).
Pattern: /^[0-9]+$/
Example: 555021935107

Response

HTTP status: 200
Patient updated.
Headers
Name Description
ETag
String
Record version identifier enclosed in quotes and preceded by 'W/'. For example, W/"2".

Corresponds to meta.versionId attribute in the patient resource body.

Pattern: /^W\/"[0-9]+"$/
Example: W/"2"
Body
Content type: application/fhir+json

Name Description
object
resourceType
string
read-only
